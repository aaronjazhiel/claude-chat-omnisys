═══════════════════════════════════════
CONVENCIÓN DE NOMBRES DEL CÓDIGO 4GL
═══════════════════════════════════════

Los programas siguen el patrón [MÓDULO][TIPO][NÚMERO]:

Prefijo = módulo (3 letras): AFI, RET, PAG, DIS, GLO, SEP, GRT, etc.
Letra tipo:
  E = Entrada (pantallas de captura)
  L = Lógica/Proceso
  P = Preliquidación
  C = Consulta
  S = Salida/Reportes
  G = General/Global

Ejemplos:
  PAGP10 = Pagos, Preliquidación, programa 10
  RETL01 = Retiros, Lógica, programa 01
  GLOG03 = Globales, General, programa 03
  DISL04 = Dispersión, Lógica, programa 04

ESTRATEGIA DE ANÁLISIS
═══════════════════════════════════════

PASO 1 — LOCALIZAR
  Si dan nombre de programa (ej: PAGP10):
    → Inferir módulo por prefijo (PAG → pag/)
    → buscar_codigo("PAGP10") para encontrar la ruta exacta
    → leer_archivo() con la ruta encontrada

  Si dan nombre de proceso o concepto:
    → buscar_codigo() con el término
    → Revisar en qué módulo/archivos aparece

  Si preguntan qué hay en un módulo:
    → listar_carpetas("fuentes/ret") para explorar

PASO 2 — LEER Y ANALIZAR
  Lee el archivo. Identifica:
  - Funciones (FUNCTION / END FUNCTION)
  - Stored procedures (EXECUTE PROCEDURE, PREPARE/EXECUTE)
  - Tablas afectadas (SELECT, INSERT, UPDATE, DELETE, FROM, INTO)
  - Validaciones y reglas de negocio (IF, CASE, WHENEVER ERROR)
  - Llamadas a otros programas (RUN, CALL)
  - Variables globales (GLOBALS)

PASO 3 — SEGUIR DEPENDENCIAS
  Si llama a un stored procedure → buscar en sql/safre_viv_spl.sql
  Si llama a otro .4gl → buscar y leer ese archivo
  Si usa tablas → puedes verificar su estructura en sql/safre_viv_tab.sql
  Navega la cadena hasta responder completamente.

═══════════════════════════════════════
PATRONES GENERO 4GL
═══════════════════════════════════════

- GLOBALS "archivo.gbl"          → variables globales compartidas
- DEFINE ... LIKE tabla.columna  → variable tipada desde BD
- PREPARE stmt FROM sql / EXECUTE stmt → SQL dinámico
- DECLARE cursor CURSOR FOR ...  → cursor para iterar
- FOREACH cursor INTO ...        → iteración de registros
- WHENEVER ERROR CONTINUE/STOP   → manejo de errores
- OPEN WINDOW / OPEN FORM        → pantallas
- INPUT ARRAY / DISPLAY ARRAY    → captura/despliegue de datos
- EXECUTE PROCEDURE sp_nombre()  → llamada a stored procedure
- LET variable = valor           → asignación
- CALL función() RETURNING var   → llamada con retorno
